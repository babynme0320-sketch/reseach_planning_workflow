# Execution Plan: Bio-R&D Dashboard UI/UX

본 문서는 `tasks.md`에 기재된 작업을 우선순위에 따라 1~2시간 규모로 구체화하여 실행 순서를 계획한 문서입니다.

## 1. 우선순위별 실행 계획

### Phase 1: 기반 구축 및 디자인 시스템 (Task 1)
- **태스크 [1.1]**: CSS 색상 시스템 및 전역 디자인 토큰 설계
  - 시간 규모: 1시간
  - 작업 내용: `index.html` 내에 프리미엄 다크/라이트 하이브리드 글래스모피즘 테마를 위한 CSS 변수(Variables) 정의 및 기본 Outfit/Inter 폰트 설정.
  - 완료 조건: 브라우저 로딩 시 폰트 및 배경 그라데이션이 미려하게 적용되고, 개발자 도구에서 CSS 변수들이 정상 인식되어야 함.
- **태스크 [1.2]**: 레이아웃 컴포넌트 폴리싱 및 사이드바 개선
  - 시간 규모: 1시간
  - 작업 내용: 사이드바 영역과 메인 콘텐츠 영역의 패딩, 마진, 스크롤바 커스터마이징, 그리고 활성 메뉴 표시 보완.
  - 완료 조건: 사이드바의 활성화된 메뉴가 눈에 띄는 백그라운드 및 보더 하이라이트를 가져야 함.

### Phase 2: 대화형 실무 툴박스 개발 (Task 2, 3, 4)
- **태스크 [2.1]**: 예산 시뮬레이터 입력 조작 및 유효성 검사 로직 구현
  - 시간 규모: 1.5시간
  - 작업 내용: 모달리티 선택 드롭다운, 예산 입력 필드, 그리고 예산 항목별 비율 조절 슬라이더 UI 구현 및 실시간 연산 JS 구현.
  - 완료 조건: 수치 입력/슬라이더 조작 시 각 예산 항목이 실시간으로 정확히 재계산되어 표시되어야 함.
- **태스크 [2.2]**: 예산 경고 및 시각 차트 업데이트
  - 시간 규모: 1시간
  - 작업 내용: 예산 배분 요율을 시각화하는 게이지바 또는 가로형 누적 바 그래프 컴포넌트를 직접 그리고, 40% 위탁비 초과 등 혁신법 위반 시 경고 알림 표시.
  - 완료 조건: 위탁비 비율이 총 예산의 40%를 넘을 때 빨간색 경고 테두리와 메시지가 동적으로 나타나야 함. (위험도: Medium - 혁신법 복잡성 반영 필요)
- **태스크 [3.1]**: 모의 자가진단 항목 구조화 및 개별 주체 체크리스트 구현
  - 시간 규모: 1.5시간
  - 작업 내용: 대학, 기업, 연구소, 병원, 총괄 등 각 역할에 부합하는 체크리스트를 접이식 아코디언(Accordion) 스타일 또는 카드 형태로 세분화하여 미려하게 구성.
  - 완료 조건: 주체별 카드가 명확하게 분류되고 접기/펼치기가 부드럽게 작동해야 함.
- **태스크 [3.2]**: 자가진단 진척도 및 합격 여부 동적 갱신
  - 시간 규모: 1시간
  - 작업 내용: 체크박스 개수 대비 선택 개수를 계산하여 상단 진단 점수 및 게이지바가 부드럽게 증가하고, 점수에 따른 평가 요약 카드 생성.
  - 완료 조건: 모든 체크박스를 다 선택했을 때 100% 진척도와 합격 문구가 보여야 함.
- **태스크 [4.1]**: TRL 로드맵 타임라인 노드 시각화 및 호버 툴팁
  - 시간 규모: 2시간
  - 작업 내용: TRL 1단계(기초연구)부터 TRL 5단계(비임상 완료)까지의 단계를 순차적인 간트차트 또는 가로형 이정표(Timeline) 노드로 렌더링. 마우스 호버 시 주관 기관 및 요건 툴팁 표시.
  - 완료 조건: 가로형 타임라인 노드가 보이고 마우스를 올렸을 때 상세 툴팁이 팝업되어야 함.

### Phase 3: 네비게이션 흐름 및 검증 (Task 5, 6)
- **태스크 [5.1]**: 마크다운 뷰어 내 상대 링크 파싱 및 뒤로가기 구현
  - 시간 규모: 1.5시간
  - 작업 내용: 마크다운 내의 `file:///` 또는 relative MD 링크 클릭 시 해당 파일을 API 서버 `/api/read`를 통해 비동기로 가져오는 이벤트 리스너 리디렉션 구현 및 상단 뒤로가기(Back) 내비게이션 추가.
  - 완료 조건: 마크다운 문서 내의 링크 클릭 시 페이지 전체 리로드 없이 메인 뷰어 영역에 해당 문서가 로드되어야 하며, 뒤로가기 버튼으로 이전 문서를 정상 복구할 수 있어야 함. (위험도: High - Marked.js 링크 가로채기 예외 처리 필요)
- **태스크 [6.1]**: 브라우저 검증 시나리오 실행 및 콘솔 검사
  - 시간 규모: 1시간
  - 작업 내용: 완성된 대시보드를 로컬 개발 서버(`server.py`) 상에서 기동한 후 Chrome과 Safari 브라우저에서 시나리오대로 테스트 실행 및 에러 확인.
  - 완료 조건: 테스트 결과와 브라우저 콘솔 캡처(로그)가 포함된 완료 레포트를 기록함.

## 2. 위험 관리 (Risk Management)
- **위험 항목 [Task 5.1 마크다운 상대 링크 인터셉트]**: Marked.js의 렌더링 방식에 따라 `href`가 절대 경로(`file:///...`)로 바뀔 때 클릭 이벤트를 잡아 가로채지 못하면 브라우저가 새 탭을 띄우거나 먹통이 되는 위험이 있습니다.
  - **대응 방안**: 렌더링 완료 직후 뷰어 콘테이너 내부의 모든 `<a>` 태그에 대해 `event.preventDefault()`가 걸린 이벤트 리스너를 일괄 바인딩하는 JS 함수를 추가 구현합니다.
