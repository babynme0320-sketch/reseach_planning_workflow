# Project Tasks: Bio-R&D Dashboard UI/UX

본 문서는 바이오 연구기획 대시보드 개선 프로젝트의 세부 태스크 목록을 관리합니다.

- **상태 정의**: `대기` (Todo) | `진행중` (In Progress) | `완료` (Done)

---

## Task 목록

### [Task 1] UI 디자인 시스템 및 비주얼 테마 전면 개편 (Aesthetic Overhaul)
- **설명**: `index.html` 내 스타일시트를 글래스모피즘(반투명 백그라운드, 드롭 섀도우) 및 프리미엄 다크/라이트 하이브리드 스타일로 전면 리모델링합니다.
- **완료 조건**:
  -Outfit/Inter 폰트가 대시보드 전체에 미려하게 적용되어야 함.
  -산학연병 주체별 고대비 색상 변수(CSS Variables)가 적용되어 컴포넌트별로 명확하게 매핑되어야 함.
  -사이드바 및 메인 프레임에 입체적인 글래스모피즘 효과가 조화롭게 표현되어야 함.
- **상태**: `완료`
- **담당**: Senior Engineer / UI Designer

### [Task 2] 혁신법 기반 예산 시뮬레이터 개선 및 시각화 (Budget Simulator)
- **설명**: 툴박스의 예산 시뮬레이터 입력 폼과 계산 결과를 시각적으로 개선하고 실시간 피드백 게이지를 추가합니다.
- **완료 조건**:
  -사용자가 총 연구비를 입력하고 모달리티 선택 시, 각 항목별 배분 비율(%)을 슬라이더나 인풋으로 미세 조율할 수 있어야 함.
  -직접비/간접비/인건비/위탁비 비율 계산 결과가 정교한 카드와 프로그레스 게이지로 실시간 시각화되어야 함.
  -혁신법 위반 사항(예: 위탁비가 직접비의 40%를 초과하는 경우) 발생 시 경고 카드와 에러 경보(Red text/border)가 나타나야 함.
- **상태**: `완료`
- **담당**: Senior Engineer

### [Task 3] 모의 자가평가 및 진단 체크리스트 고도화 (Self-Diagnosis Checklist)
- **설명**: 사용자가 작성된 연구계획서 요건을 점검할 수 있는 자가평가 진단 도구를 동적으로 개선합니다.
- **완료 조건**:
  -기업(시장성), 연구소(데이터 Rigor), 학교/병원(생명윤리), 총괄(예산) 각 주체별 체크박스가 더 미려하고 구조화되어 표현되어야 함.
  -항목을 체크할 때마다 전체 진척도가 실시간으로 계산되어 상단 프로그레스 바(Progress Bar)와 총점에 반영되어야 함.
  -점수에 따른 진단 결과 요약 문구(예: "미흡", "양호", "우수 (평가위원 통과권)")가 역동적으로 출력되어야 함.
- **상태**: `완료`
- **담당**: Senior Engineer / QA

### [Task 4] TRL 로드맵 및 연차별 마일스톤 시각화 (TRL Roadmap)
- **설명**: TRL 로드맵 영역을 단순 줄글 텍스트에서 1단계부터 5단계까지의 흐름을 보여주는 반응형 타임라인 또는 간트차트 형태로 구현합니다.
- **완료 조건**:
  -각 TRL 단계별(1~5단계) 상세 기획 조건이 시각화된 노드 형태로 렌더링되어야 함.
  -단계별 카드 마우스 호버 시 툴팁(Tooltip)을 제공하여 필요 요건과 주관 기관(학교, 연구소, 병원 등)이 부각되어야 함.
- **상태**: `완료`
- **담당**: Senior Engineer

### [Task 5] 네비게이션 및 마크다운 뷰어 흐름 최적화 (User Flow)
- **설명**: 사이드바 메뉴 클릭 시 화면 전환 사용자 흐름을 개선하고, 마크다운 문서 뷰어의 편의성을 극대화합니다.
- **완료 조건**:
  -사이드바에서 선택된 항목이 하이라이트되고, 메인 영역의 로딩 상태가 매끄럽게 처리되어야 함.
  -마크다운 뷰어 내의 상대 경로 파일 링크 클릭 시 백엔드 API를 통해 다음 마크다운 문서를 자연스럽게 불러오는 "상대 이동 흐름"이 오차 없이 작동해야 함.
  -이전 문서로 돌아갈 수 있는 "뒤로가기" 버튼이 메인 뷰어 상단에 제공되어야 함.
- **상태**: `완료` (Claude: 누락 JS 헬퍼 복원 + `/api/read` 마크다운 뷰어 동작 확인)
- **담당**: PM / Senior Engineer

### [Task 6] 화면 검증 및 크로스 브라우저 테스트 (Verification & Test)
- **설명**: 완성된 대시보드가 정상 동작하는지 브라우저에서 최종 테스트를 실행하고, 그 검증 결과를 기록합니다.
- **완료 조건**:
  -Chrome 및 Safari 브라우저에서 대시보드를 열어 레이아웃과 툴 기능이 정상 작동함을 스크린샷이나 브라우저 테스트 로그로 검증해야 함.
  -브라우저 개발자 도구 콘솔에 JS Runtime Error 로그가 없음을 입증해야 함.
- **상태**: `완료` (Claude: 서버 스모크 테스트 통과 — index.html HTTP 200, JS 문법/함수 정의 검증, /api/read 200, 경로이탈 404 차단. ⚠️ Chrome/Safari 육안 렌더링·콘솔 확인은 사용자 최종 확인 권장)
- **담당**: QA Reviewer
