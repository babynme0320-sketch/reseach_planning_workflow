# Success Criteria: Bio-R&D Dashboard UI/UX

본 문서는 바이오 연구기획 대시보드의 성공 기준 및 검증 지표를 정의합니다.

## 1. 정량적 성공 기준 (Quantitative Metrics)
- **콘솔 오류 Zero**: 대시보드 조작 및 파일 로딩 시 브라우저 개발자 도구 콘솔에 JavaScript 런타임 에러(`Uncaught ReferenceError`, `TypeError` 등)가 전혀 발생하지 않아야 함.
- **예산 실시간 계산 오차 0%**: 예산 시뮬레이터에서 총액 변경 및 비율 선택 시, 계산된 직접비/간접비/인건비의 합산 수치가 산술적으로 정확하게 떨어져야 함.
- **크로스 브라우저 호환성**: macOS의 Chrome, Safari 브라우저에서 모두 동일하게 레이아웃이 렌더링되고 작동해야 함.

---

## 2. 정성적 성공 기준 (Qualitative Metrics)

### 2.1 UI 디자인 및 프리미엄 비주얼
- **사용자 첫인상**: 단순히 흰 배경에 검은 텍스트(기본 스타일)가 아닌, 세련된 그라데이션 테두리, 일관성 있는 섀도우, 다크/라이트 하이브리드 글래스모피즘 카드가 눈에 띄게 드러나야 함.
- **가독성**: 마크다운 파일(예: 핸드북 챕터)을 불러왔을 때 폰트 패밀리(Outfit/Inter), 글자 크기, 행간, 표(table) 디자인이 매우 깔끔하게 정리되어 읽기 편해야 함.

### 2.2 사용자 흐름 및 버튼 배치
- **네비게이션**: 사용자가 사이드바 메뉴를 클릭할 때, 클릭한 아이템에 부드러운 하이라이트 효과가 적용되고 즉시 우측 메인 영역에 문서나 도구가 표시되어야 함.
- **조작 반응**: 예산 변경 슬라이더/인풋 조작 시, 혹은 자가진단 항목 체크 시 0.1초 이내에 시각적 그래프나 피드백 카드가 자연스러운 애니메이션과 함께 변경되어야 함.

---

## 3. 검증 방법 (Verification Methods)
- **수동 기능 검증**: 사이드바 메뉴 10개 이상 클릭 테스트, 예산 시뮬레이터 수치 입력 테스트, 체크리스트 전체 선택 테스트.
- **브라우저 콘솔 로그 검사**: 개발자 도구 콘솔에 빨간색 에러 로그 유무 검증.
- **화면 반응성 및 스크롤 검증**: 창 크기를 줄였을 때(태블릿 규격) 사이드바와 콘텐츠가 깨지지 않고 자연스럽게 스크롤되거나 리플로우되는지 검증.
