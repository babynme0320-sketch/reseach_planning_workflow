# 바이오 R&D 기획 기밀 보안 및 데이터 무결성 지침 (Bio-R&D Security Guide)

본 지침은 국가 R&D 과제 기획 과정에서 도출되는 후보물질 아미노산 서열, 화학 구조식, 특허 출원 전 선행 데이터, 동물 실험 및 환자 유래물 데이터의 유출을 방지하고 무결성을 유지하기 위한 정보 보안 프레임워크를 규정합니다.

---

## 1. 지식재산권(IP) 보호 및 특허 보안

### 1.1 미공개 특허 출원 정보 관리
- **원칙**: 출원서 제출 전인 핵심 아미노산 서열, 핵산 서열, 타겟 바인더 구조 정보는 계획서 초안 내에 평문으로 기재하지 않는 것을 권장함.
- **조치**: 필요 시 서열 정보를 난독화하여 표기하고, 최종 제출 직전 주관 연구책임자(PI) 검토 단계에서만 원본 서열을 병합함.
- **FTO 사전 검토 기록**: 특허 침해 회피성(FTO) 분석 보고서는 기밀 문서로 분류하여 로컬 가상 환경 외의 공용 서버나 승인되지 않은 외부 LLM API로 송신하지 않음.

### 1.2 연구원 비밀유지 계약 (NDA)
- **대상**: 과제 기획에 참여하는 주관/공동 연구원 및 협업 에이전트.
- **수칙**: 과제 미선정 시 기획 단계에서 도출된 아이디어와 선행 실험 결과는 즉시 폐기하거나 아카이빙 처리하여 타 프로젝트로의 무단 도용을 원천 차단함.

---

## 2. 생명 정보 및 환자 유래 데이터 보안

### 2.1 개인정보 및 유전정보 익명화 (Bio-Data Obfuscation)
- **대상**: 인간 유래물(혈액, 조직 등)을 이용한 진단/치료제 개발 과제 기획 시.
- **가이드라인**: 환자 대상의 모든 오믹스(Genomics, Transcriptomics 등) 데이터 및 임상 기록은 생명윤리법에 의거하여 **완전 익명화(De-identification)** 처리된 식별자만 기재함.
- **IRB 연동**: 생명윤리심의위원회(IRB) 승인서 상의 승인 번호 및 프로토콜 규격을 초과하는 세부 환자 개인정보의 계획서 삽입을 엄격히 금지함.

---

## 3. 물리적 및 시스템 정보 보안 통제 (System Security Gates)

### 3.1 파일 관리 및 Git 커밋 게이트
- **버전 관리 차단 (.gitignore 가이드)**:
  - 후보물질 서열 원본, GLP 기관에서 수령한 시험 견적서 PDF, 원본 이미지 로그 데이터는 Git 추적 대상에서 완전히 배제함.
  - `.env` 파일에 저장된 OpenRouter/NCBI API 키 유출을 차단하기 위한 사전 커밋 훅(Pre-commit Hook) 작동을 의무화함.
- **외부 유출 통제**:
  - `source-papers/` 폴더 내에 저장되는 모든 특허 원본 및 미발표 논문 초안은 외부 네트워크 유출 방지(DLP) 경로 내에서만 보관함.
